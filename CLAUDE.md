# CLAUDE.md

## プロジェクト概要

ARG・謎解き企画向けの**疑似SNS基盤**。GitHub Pages で静的配信する参照専用SNSビューアと、データ生成ツール（generator）で構成される。

対象ユーザー：非エンジニアのARG企画者・ジュニアエンジニア（fork & JSON編集が前提）

## アーキテクチャ

### viewer（フロントエンド）

- HTML + Vanilla JS + 素のCSS（フレームワーク・ビルドツール一切不使用）
- クライアントサイドのみで動作する参照専用ビューア
- 検索はフロントエンドでJSONを全文走査（インデックス不使用 — ARG解析耐性のため意図的）
- レスポンシブ必須（PC・スマホ両対応）
- CSS変数・media queryで実装

### data

- `data/{userKey}/user.json` にユーザー情報＋投稿（posts配列）を格納
- 投稿日時は基準日からの相対時間で保持（config.json で固定日 or 閲覧日基準を選択）
- フォロー/フォロワー関係の整合性は厳密に求めない（viewer側でフォールバック表示）

### generator（Node.js）

2系統に明確分離：

1. **ユーザー作成 generator** — フォルダ作成・user.json最小生成のみ。投稿生成はしない。同一keyが存在する場合はエラー（上書き不可）
2. **ポスト追加 generator** — 既存user.jsonに追記のみ（削除・上書きしない）。sampleパターンから抽選・シャッフル・時刻ずらしで生成

## 重要な設計方針

- **破壊的操作を避ける** — generatorは常に追記。既存データを消さない
- **手動編集ルートを必ず残す** — GitHub Actionsに完全依存させない
- **ビルド不要を維持** — Tailwind・バンドラー等を導入しない
- **難読化はオプション** — 完全秘匿ではなく「解析しても意味が出ない構造」を重視
- **sample解析しても答えが出ない** — sampleはテンプレート素材であり、実際の謎の答えを含まない

## ファイル編集指針

- `viewer/` を変更する場合は必ずPC・スマホ両方で表示確認を意識する
- `generator/` の変更は「破壊的操作をしない」原則を守る
- `config.json` のスキーマを変更した場合はREADMEも更新する
- `data/` 配下のJSONスキーマを変更した場合はsampleデータとの整合性を確認する

## 制約

- Node.js は generator のみで使用（viewer側には持ち込まない）
- 外部CDN・npm依存はviewer側に追加しない
- GitHub Pages の容量制限：合計30MB以内を目安
